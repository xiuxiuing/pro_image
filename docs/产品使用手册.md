# 商品竞对分析平台 · 使用手册

面向日常操作：从启动、授权到项目创建、工作台比对与导出。打包安装与 Nuitka 等部署细节见仓库根目录 `packaging_nuitka.md` 等文档。

---

## 一、产品能做什么

**商品竞对分析平台**用于对比「主店」与多家「竞店」的商品数据，典型用途包括：

- 自动或辅助完成 **SKU 与竞店商品的匹配**（含向量分析与人工校准）。
- 在同一行横向对比 **主店与各竞店** 的价格、销量、图片与文本字段。
- 对单行做 **跟价 / 取消跟价**、**参考名称或参考主图**、**取消关联 / 手动选品关联**、**标记新增**、**淘汰与已处理** 等操作。
- 将当前项目结果 **导出为文件**（全量打包与「修改/新增」专项导出）。

店铺列名来自您上传的 Excel 文件名（去掉 `.xlsx` / `.xls` 后缀）。

---

## 二、授权与首次打开（重要）

软件受 **离线授权** 保护。若浏览器打开后提示 **「软件未激活」**：

1. 页面上会显示本机的 **机器指纹 (HWID)**，可点击 **复制机器指纹**。
2. 将 HWID 发给供应商，获取授权文件。
3. 将授权文件保存为程序运行目录下的 **`license.dat`**（与源码运行时项目根目录）。
4. 刷新页面或重新启动程序。

若授权无效或缺失，无法进入 **项目列表** 与 **工作台**。

## 三、如何启动（源码运行）

1. 在电脑已安装 **Python 3** 的前提下，进入项目根目录。
2. 安装依赖（若尚未安装）：
   ```bash
   pip install -r requirements.txt
   ```
3. 启动服务：
   ```bash
   python3 app.py
   ```
4. 浏览器访问：**http://127.0.0.1:5001**（默认端口 **5001**）。

首次启动若加载 AI 相关模型，可能需等待片刻。结束使用时在运行 `app.py` 的终端中按 **Ctrl+C** 停止。

**说明：** 若使用供应商提供的 **已打包版本**（如 `.app`），启动方式以交付说明为准；部分版本会在启动后自动尝试打开浏览器。

## 四、项目列表页：创建与进入项目

### 4.1 新建项目

点击 **「+」** 卡片打开 **「初始化分析项目」** 对话框，需填写：

| 项目 | 说明 |
|------|------|
| **项目名称** | 必填，用于区分不同批次分析。 |
| **主店文件** | 必填，单个 Excel：**`.xlsx` 或 `.xls`**。 |
| **竞店文件** | 必填，至少一个；可多选多个文件，每个文件对应一家竞店。 |
| **文件大小** | 单个文件建议不超过约 **80MB**；一次请求总上传上限约 **100MB**（超出会提示错误）。 |

### 4.2 两种分析模式（二选一）

- **自动 AI 分析（默认）**  
  - 上传主店与竞店源表后，由系统在后台完成 **AI 提取（可选）→ 向量分析 → 查询匹配** 等步骤。  
  - **AI 智能属性提取（可选）**：勾选后需填写 **Gemini API Key**；可选用 **Gemini 模型名称**（留空则走内置默认）。用于补齐品牌、规格等属性，**推荐在数据质量一般时开启**（会产生 API 调用费用，以您的密钥与计费为准）。  
  - 未勾选 AI 时，仍可进行后续向量与匹配流程（取决于源表字段是否满足分析要求）。

- **跳过分析（手动上传）**  
  - 适用于您已有 **离线算好的比对结果 Excel**。除主店、竞店源文件外，必须再上传 **比对结果文件**（界面中 **「手工结果」**）。  
  - 提交后会 **直接导入结果** 并进入可用状态，**不跑** 自动分析流水线。

### 4.3 项目状态与操作

- **分析中**：卡片显示进度状态，可 **进度详情** 查看各步骤（AI 提取、向量分析、查询匹配等）与大致进度；分析耗时与数据量、是否启用 AI 有关。若长时间无进展，请保持电脑 **不休眠、不关机**，并避免同时跑过多任务。  
- **可进入（就绪）**：显示创建时间与 **SKU 数量**，可 **进入工作台**。  
- **分析失败**：需 **删除** 该项目后，检查源表与网络（若使用 AI）再重新创建。

列表会 **定时刷新**；若浏览器允许 **桌面通知**，分析完成或失败时可能收到提示。

## 五、工作台（数据看板）概览

从项目列表 **进入工作台** 后，地址为 **`/dashboard`**。页头显示 **当前项目名称**；点击标题区域可 **返回项目中心**（`/`，即项目列表）。

主体为 **横向多列布局**：最左 **主店** 一列，右侧每个竞店一列，同一 **主店 SKU 一行** 对齐展示，便于对比价格、销量与图文信息。

## 六、搜索、筛选、排序与列设置

### 6.1 顶部搜索

- **搜索框**：按 **标题 / SKUID** 等关键字过滤（与后端分页查询一致）。  
- 输入后可通过 **清除按钮** 快速清空并重新加载。

### 6.2 「展示筛选」面板

点击 **「展示筛选」** 打开浮层，可：

- **搜索字段名**：快速定位某一列。  
- **显示/隐藏列**：勾选要在卡片上展示的字段；**主图链接** 常作为主图展示依据。  
- **拖拽排序**：左侧拖动调整 **列顺序**（与展示顺序一致）。  
- **全选 / 全不选 / 恢复默认顺序**：批量操作与恢复初始列配置。  
- **每列「筛选…」**：在对应列上做 **关键词包含** 类过滤（与全局搜索叠加）。  
- **每列 ↑ / ↓**：按该列 **升序 / 降序** 排序（与主店表头中的筛选模式配合使用）。

### 6.3 主店表头上的快捷筛选

在 **主店列标题** 区域提供：

- **无关联：开/关**：仅查看 **尚未与任一竞店建立关联** 的主店商品（便于优先补关联）。  
- **未处理：开/关**：仅查看 **未标记为已处理** 的行（与「已处理」标记配合做进度管理）。

**说明：** 「无关联」与「未处理」两种模式 **互斥**，开启其一会自动关闭另一项。

### 6.4 竞店表头上的销量排序

各 **竞店列标题** 上可对 **「该店销量」** 字段做 **升序 / 降序**，便于快速发现高低销量商品。

### 6.5 「未关联数据」模式

顶部 **「未关联数据」** 按钮用于切换到 **竞店侧未关联商品池**视图（与主店列表视图不同）：用于浏览、搜索并 **把竞店商品关联到主店 SKU**（见下文「手动关联」）。按钮激活时文案可能变为 **「关联数据」**，再次点击可切回普通视图。

## 七、主店卡片与竞店卡片上常见操作

### 7.1 主店（左侧）

- **已处理**：标记该行是否已跟进；勾选后可在「未处理」筛选中隐藏。  
- **淘汰 / 恢复**：将商品标记为淘汰或恢复（具体以界面按钮为准）。  
- **主图**：支持维护主图（以页面实际能力为准）。

### 7.2 竞店（各竞店列）

常用图标按钮（以页面实际提示为准）：

| 操作 | 含义 |
|------|------|
| **参考名称 / 参考图片** | 指定将 **哪一家竞店** 的名称或图片作为参考展示（与主店信息对比时）。 |
| **取消关联** | 解除当前主店 SKU 与该竞店商品的匹配关系。 |
| **手动选择** | 打开弹窗，从该竞店商品列表中 **搜索并选择** 应对应的商品，完成 **人工关联**。 |
| **智能跟价 / 取消跟价** | 按规则计算并写入跟价相关字段，或清除跟价；**已跟价** 时列上可能显示 **跟价店** 标签。 |
| **标记新增** | 标记该竞店侧为「新增」类商品（用于后续统计与导出）。 |

空位（无匹配）时，通常可 **手动选择** 从竞店商品池建立关联。

### 7.3 复制与剪贴板

部分字段（如商品名称、skuId、美团类目等）支持 **点击复制**；若浏览器拦截剪贴板权限，请在站点设置中 **允许剪贴板**，否则复制可能失败。

## 八、导出

| 按钮 | 说明 |
|------|------|
| **全部导出** | 导出当前项目下 **完整结果**（后端可能打包为 **ZIP**，内含多个表格文件，文件名以日期为准）。 |
| **修改导出** | 导出与 **修改/新增** 相关的数据（通常为 **Excel**），便于单独提交或核对。 |

导出时按钮会短暂显示「正在生成…」，请勿重复猛点；若失败可刷新页面后重试。

---

## 九、使用建议

- 数据量大时，请 **优先使用分页、列筛选与「无关联 / 未处理」**，避免一次渲染过多行导致浏览器卡顿。  
- **跟价、参考图/名称、取消关联、手动关联** 会 **改库内业务数据**，操作前请确认当前行与店铺列无误。  
- 多项目 **同时分析** 会拖慢速度，界面也会提示；建议错峰创建。  
- 使用 **Gemini AI 提取** 时，请妥善保管 **API Key**，勿提交到公开仓库。

## 十、常见问题

**打不开页面？**  
确认本机已执行 `python3 app.py` 且无报错，浏览器访问 `http://127.0.0.1:5001`；并确认已通过 **授权**（存在有效 `license.dat`）。

**端口被占用？**  
关闭占用 **5001** 端口的其他程序；或修改 `app.py` 末尾的 `app.run(..., port=5001)` 为其他端口，并改用新地址访问（需一定技术基础）。

**上传失败或提示体积过大？**  
检查单文件是否超过约 **80MB**、多文件总和是否超过约 **100MB**；必要时拆分 Excel 或减少单次上传量。

**分析一直转圈？**  
保持电脑不休眠；若超过较长时间仍无结果，可到 **进度详情** 查看卡在哪一步；失败项目需 **删除后重建**。

**数据未刷新？**  
操作完成后可 **刷新页面** 或 **重新进入项目**；若处于「未关联数据」等特殊视图，切换视图或重新加载列表。

**激活页与项目页的关系？**  
未激活时只能看到 **HWID 与激活说明**；激活成功后进入 **项目列表**。
